# 第 8 阶段：测试、修复与演示数据准备

本文档用于《软件体系结构》课程项目答辩与报告整理，覆盖系统启动、数据库初始化、演示数据、测试表、已修复问题和答辩演示顺序。

## 1. 系统启动说明

### 后端启动

```powershell
cd D:\软件体系结构\backend
$env:JAVA_HOME='C:\Program Files\Java\jdk1.8.0_192'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn spring-boot:run
```

后端默认地址：

```text
http://localhost:8080
```

也可以先打包再运行：

```powershell
cd D:\软件体系结构\backend
$env:JAVA_HOME='C:\Program Files\Java\jdk1.8.0_192'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn -DskipTests package
java -jar target/coursework-system-backend-0.0.1-SNAPSHOT-boot.jar
```

### 前端启动

```powershell
cd D:\软件体系结构\frontend
npm install
npm run dev
```

前端默认地址：

```text
http://localhost:5173
```

## 2. 数据库初始化说明

先在 MySQL 中创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS coursework_system
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

检查 `backend/src/main/resources/application.yml` 中的数据库账号、密码：

```yaml
spring:
  datasource:
    username: root
    password: 123456
```

后端启动时会自动执行 `backend/src/main/resources/db/init.sql` 创建基础表。`DataInitializer` 会初始化角色和基础账号，`DemoDataInitializer` 会初始化第 8 阶段演示课程、作业、提交、重复检测、批改和通知数据。

演示文件默认写入后端运行目录下的：

```text
uploads/demo
```

Excel 导出文件默认写入：

```text
exports
```

## 5. 演示流程

1. 使用 `admin` 登录，查看管理员菜单：课程审批、用户管理、系统日志。
2. 使用 `teacher01` 登录，查看教师菜单：我的课程、课程申请、作业管理、成绩统计。
3. 在“我的课程”中查看演示课程“软件体系结构”。
4. 进入课程成员管理，确认 `assistant01`、`student01`、`student02`、`student03` 已加入课程。
5. 进入作业管理，查看四类演示作业。
6. 使用 `student01` 登录，进入“我的作业”，查看作业并查看已有提交。
7. 查看 `student01` 的单选题提交，确认自动判分结果。
8. 使用 `student02` 登录，查看迟交状态和判断题自动 `0` 分结果。
9. 使用 `teacher01` 登录，进入文件作业的疑似重复页面，查看同 Hash 的 `student01` 与 `student03` 记录。
10. 进入批改列表和批改详情页，查看迟交、疑似重复、自动判分和人工评分区域。
11. 使用学生账号进入“我的成绩”，查看最终成绩和评语。
12. 进入通知中心，查看并标记通知已读。
13. 使用教师账号进入“成绩统计”，查看统计指标和图表。
14. 点击导出 Excel，检查导出表字段完整。

## 9. 仍存在的问题

1. 当前环境已完成后端测试和前端构建验证，但未在浏览器中逐页完成全量人工点击录屏；答辩前建议按本文档流程再走一遍并截图。

3. Excel 导出当前为同步导出，适合课程项目和演示数据；生产级大数据量建议改为异步导出任务。
4. 已有旧数据库如果表结构早于阶段 7，建议重新建库或确认 `notifications` 的 `receiver_id`、`read_status` 字段已存在。
5. 非功能测试中的响应时间需要在实际演示机器和数据库环境中补充截图或记录数值。

## 11. 答辩演示顺序

1. 介绍项目定位：课程作业提交与批改管理系统，不使用 AI，“智能”来自规则和事件。
2. 展示登录与 RBAC：四类账号登录后菜单不同，后端接口也做权限校验。
3. 展示课程流程：教师申请、管理员审批、教师管理成员。

6. 展示规则系统：迟交判断、客观题自动判分、同作业全班文件 Hash 疑似重复。
7. 展示批改系统：教师或助教批改，疑似重复只提示不自动判 0 分。
8. 展示学生成绩：学生只能查看自己的成绩和评语。
9. 展示通知中心：申请、审批、发布、提交、批改、退回等事件通知。
10. 展示统计和 Excel 导出：课程统计、作业统计、图表、成绩表导出。
11. 总结体系结构风格：层次系统、面向对象、仓库、规则系统、事件系统、管道-过滤器、批处理。
