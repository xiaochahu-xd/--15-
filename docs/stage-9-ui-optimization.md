| 接口 | 说明 |
| --- | --- |
| `GET /api/dashboard` | 根据当前用户角色返回工作台聚合数据 |

## 6. 新增后端接口

新增接口同第 5 节，均为只读聚合接口或通知计数接口，不改变原有业务主流程。

## 7. 通知逻辑实现

1. 继续复用 `notifications` 表。
2. 通知中心通过 `GET /api/notifications` 获取当前用户通知。
3. 顶部铃铛通过 `GET /api/notifications/unread-count` 获取未读数。
4. 单条已读调用 `PUT /api/notifications/{id}/read`。
5. 全部已读调用 `PUT /api/notifications/read-all`。
6. 点击通知后按通知类型跳转到课程审批、课程列表、作业列表、批改页或成绩页。
7. 后端事件系统仍负责课程申请、审批、作业发布、提交、批改、退回等业务通知生成。

## 8. 课程详情页进入方式

1. 从“我的课程”“负责课程”“课程管理”卡片点击“进入课程”。
2. 直接访问 `/courses/:courseId`。
3. 学生、助教、教师只能进入自己加入或授权的课程。
4. 管理员可进入所有课程。
5. 后端 `GET /api/courses/{courseId}/detail` 会校验当前用户课程访问权限。

## 9. 作业类型筛选实现

1. 作业管理页使用前端 `typeFilter` 记录当前筛选项。
2. 筛选项包括：全部、文本作业、文件作业、单选题作业、判断题作业。
3. 筛选只影响作业列表，不影响发布作业表单。
4. 统计卡片基于当前课程作业列表计算。
5. 课程详情页作业 Tab 也支持同样类型筛选。

## 10. 作业状态标签计算

当前按截止时间计算：

| 条件 | 标签 |
| --- | --- |
| 当前时间距离截止时间小于等于 24 小时且未截止 | 即将截止 |
| 当前时间超过截止时间 | 已截止 |
| 其他未截止作业 | 进行中 |

迟交和疑似重复状态在提交详情、课程详情提交 Tab、待批改列表和我的提交页中单独显示。

## 11. 待批改筛选实现

待批改列表支持：

1. 全部。
2. 待批改。
3. 已批改。
4. 迟交。
5. 疑似重复。
6. 文本作业。
7. 文件作业。
8. 客观题。
9. 自动判分。
10. 已退回。

筛选由前端基于 `gradingItems` 本地计算，后端权限仍由批改接口和课程成员校验保证。

## 16. 当前仍存在的问题

1. 通知表当前没有统一的业务对象跳转字段，因此通知点击跳转按通知类型推断到对应模块入口，不能精确跳到某一条作业或课程。
2. 课程详情页中的课程通知 Tab 目前展示当前用户通知列表，尚未按课程 ID 严格过滤。
3. ECharts 仍会带来前端 chunk 体积提醒，不影响演示。
4. 大规模数据下“学生我的提交”聚合页会按课程和作业逐个请求，演示数据没问题，生产环境建议增加后端聚合接口。
