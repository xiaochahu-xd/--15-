## 19. 适合写入课程报告的系统实现总结

本系统采用前后端分离实现，前端使用 Vue 3、Vite、TypeScript 和 Element Plus，后端使用 Spring Boot、Spring Security、JWT、MyBatis Plus 和 MySQL。系统围绕课程作业提交与批改业务建立用户、角色、课程、课程申请、课程成员、作业、题目、提交、文件、成绩、通知、日志等核心对象，并通过数据库作为中心仓库存储业务状态。

后端整体采用层次系统风格，Controller 负责接收请求，Service 承载业务规则，Mapper 访问数据仓库，Entity 和 DTO 表达业务对象和传输对象。权限控制采用 RBAC 与课程成员授权结合的方式，既在接口入口限制角色，也在服务层校验课程归属，避免仅依赖前端菜单隐藏。

系统的“智能”能力由独立规则系统实现。规则模块通过 `Rule`、`RuleContext`、`RuleResult` 和 `RuleEngine` 组织迟交判断、单选题判分、判断题判分和文件 Hash 疑似重复检测。规则逻辑不写在 Controller 中，便于独立测试和后续扩展多选题、迟交扣分、成绩等级等规则。

事件系统通过 `SystemEvent`、`EventManager`、`EventHandler` 和多个事件处理器实现业务解耦。课程申请、课程审批、作业发布、作业提交、批改完成、作业退回等业务动作会发布事件，通知处理器生成站内通知，日志处理器记录操作日志，统计处理器作为扩展点预留。事件处理失败不会影响核心业务数据保存。

作业提交流程体现管道-过滤器思想：身份与课程权限校验、提交记录创建、文件保存与 Hash 计算、规则引擎执行、成绩记录更新、重复记录保存、事件通知依次完成。成绩导出体现批处理思想：批量读取课程成绩、汇总学生与作业信息、生成 Excel、保存导出记录并返回下载文件。整体系统满足课程项目对层次系统、面向对象、仓库、规则系统、事件系统、管道-过滤器和批处理风格的报告组织要求。
