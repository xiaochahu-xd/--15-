# 答辩演示指南

## 1. 启动方式

### 后端

```cmd
cd /d "D:\软件体系结构\backend"
set JAVA_HOME=C:\Program Files\Java\jdk1.8.0_192
set PATH=%JAVA_HOME%\bin;%PATH%
mvn spring-boot:run
```

后端地址：`http://localhost:8080`

### 前端

```cmd
cd /d "D:\软件体系结构\frontend"
npm run dev
```

前端地址以 Vite 输出为准，通常是 `http://localhost:5173`。

## 2. 数据库初始化方式

先进入 MySQL 客户端，不要在 CMD 里直接输入 SQL：

```cmd
D:\Tools\mysql\mysql-8.0.46-winx64\bin\mysql.exe -u root -p
```

进入 MySQL 后执行：

```sql
CREATE DATABASE IF NOT EXISTS coursework_system
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_unicode_ci;
```

启动后端时，Spring Boot 会执行 `backend/src/main/resources/db/init.sql` 建表，并由初始化类准备演示账号和演示数据。

## 4. 5-7 分钟演示流程

1. 管理员登录，展示管理员菜单、课程审批、系统日志，说明后端 RBAC 权限校验。
2. 教师登录，展示课程申请和“软件体系结构”课程详情，说明教师申请、管理员审批、教师管理成员。
3. 进入课程详情，展示成员、作业、提交、统计、通知五类课程内视图。
4. 教师展示四类作业：文本作业、文件作业、单选题作业、判断题作业。
5. 学生登录，展示我的课程、我的作业、我的提交，提交文本/文件/客观题答案。
6. 展示规则系统效果：迟交标记、单选题自动判分、判断题自动判分、文件 Hash 疑似重复。
7. 教师或助教进入待批改列表，展示主观题批改、最终成绩、评语、退回修改。
8. 学生查看成绩和评语，展示通知中心中的批改通知。
9. 教师展示课程统计、作业统计、提交率、迟交率、分数分布，并导出 Excel。
10. 最后打开 `docs/final-test-report.md`，总结功能测试、权限测试、规则测试、非功能测试结果。

## 6. 报告可写总结

本系统采用分层结构组织后端代码，Controller 负责接口适配，Service 负责业务流程，Mapper 负责数据访问，MySQL 作为中心仓库保存课程、作业、提交、成绩、通知等核心状态。系统将迟交判断、客观题判分、文件 Hash 重复检测抽象为独立规则模块，由 RuleEngine 统一调度，体现规则系统风格。课程申请、作业发布、作业提交、批改完成、退回修改等动作通过 EventManager 发布系统事件，再由 NotificationHandler、LogHandler 等处理器完成通知和日志，体现事件系统风格。作业提交过程按照权限校验、数据保存、文件处理、规则执行、事件通知和日志记录顺序处理，可解释为管道-过滤器风格。成绩 Excel 导出按照读取、汇总、生成文件、记录日志、返回下载的批处理流程实现。
